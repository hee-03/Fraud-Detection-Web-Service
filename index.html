API 명세

인증 방식: Session-Based Authentication (Flask session) · 응답 포맷: JSON (AI 분석 결과, SHAP 이미지 Base64 포함) · 에러 처리: 권한 없음 → 로그인 페이지 리다이렉트, 모델 미로드 → {"error": "모델이 로드되지 않았습니다."}
POST /login 은행코드 / 아이디 / 비밀번호 인증
GET /main_admin 관리자 대시보드 관리자 전용
GET /main_bank 은행 대시보드 — bank_code 기반 데이터 필터 은행 전용
POST /admin_data_input 거래 데이터 입력 및 AI 이상거래 분석 요청
GET /error_detail 이상거래 고객 상세 데이터 조회
GET/POST /boardList/<board_type> 게시판 목록 조회 · 등록 · 수정 · 삭제 (CRUD)
WS /chat WebSocket 실시간 채팅 (관리자↔은행, 동일 은행 내)
GET/POST /scheduler 일정 CRUD + 출퇴근 기록 관리

개발 중 주요 이슈 & 해결 - 트러블슈팅

📁
파일 업/다운로드 경로 불일치
원인

파일 저장 경로를 절대 경로로 하드코딩하여 팀원마다 로컬 경로가 달라 파일 기능이 정상 작동하지 않음

해결

프로젝트 루트 기준 상대 경로로 전면 변경

UPLOAD_FOLDER = os.path.join(os.path.dirname(__file__), 'static/uploads')
성과
✅ 모든 개발 환경에서 파일 업로드·다운로드 정상 동작
🔑
은행코드 PK 중복 제약으로 동일 은행 다중 계정 생성 불가
원인

bank_code 컬럼에 PK 제약 설정으로 같은 은행 소속의 여러 직원 계정을 만들 수 없음

해결

bank_code 컬럼의 PK 제약 해제, 복합 키 또는 별도 member_id PK로 DB 재설계

성과
✅ 동일 은행 소속 복수 계정 정상 등록 가능
🔓
로그아웃 후 관리자 대시보드 재진입 가능 (세션 미삭제)
원인

로그아웃 시 세션이 완전히 삭제되지 않아 URL 직접 접근으로 관리자 페이지에 재진입 가능

해결

로그아웃 라우트에서 session.clear() 명시적 호출 및 모든 보호 라우트에 @login_required 데코레이터 적용

session.clear() # 로그아웃 라우트 @login_required # 모든 보호 라우트
성과
✅ 로그아웃 후 보호 페이지 접근 시 로그인 페이지로 강제 리다이렉트
🤖
AI 모델 파일 미존재로 신규 환경에서 서버 기동 실패
원인

Git에 학습된 모델 파일(.pkl)이 포함되지 않아 새 환경에서 바로 실행하면 모델 로드 오류 발생

해결

README에 최초 실행 전 모델 학습·저장 스크립트 실행 필수 명시, 모델 미존재 시 예외 처리 추가

실행 / 배포 가이드

1
의존성 설치

프로젝트에 필요한 Python 패키지를 설치합니다.

pip install flask oracledb flask-mail xgboost shap pandas scikit-learn joblib matplotlib
2
DB 설정

Oracle DB 연결 정보를 config.py에 입력 후 DDL 스크립트를 실행하여 테이블을 생성합니다.

# config.py 에 Oracle 연결 정보 입력 # DDL 스크립트 실행으로 테이블 생성 FFG_MEMBER, FFG_TX_DATA, FFG_BOARD, FFG_CHAT, FFG_SCHEDULE
3
AI 모델 학습 (최초 1회 필수)

학습 스크립트를 실행하여 model.pkl을 생성합니다. Git에는 .pkl 파일이 포함되지 않으므로 반드시 먼저 실행해야 합니다.

python train_model.py # → model.pkl 생성 완료
4
서버 실행

Flask 개발 서버 또는 flask 명령으로 실행합니다.

python app.py # 또는 flask run --host=0.0.0.0 --port=5000
5
접속 및 로그인

브라우저에서 접속 후 관리자 계정(은행코드: 01)으로 로그인합니다.

http://ip:5000/login # 관리자 계정: 은행코드 01
